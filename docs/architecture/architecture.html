06

v0.1 → v0.2 事务

唯一允许永久移除旧活动 Outline 的路径。

inspect_v01_vault()

只读识别旧 cache、outline、trash 和阻断项。

PREFLIGHT
copy full Vault

备份根位于活动 Vault 外；失败即停止。

BACKUP
convert in staging

旧 raw → 初稿副本 + Summary；notes → 首条 Highlight；title → legacy_title。

STAGE
read every staged Paper

数量、Summary、初稿副本和报告全部验证。

VERIFY
swap staged Vault

全部成功后才替换活动 cache，并移除活动旧 Outline。

SWAP
rebuild index + retain report

备份仍是人工回滚来源；应用不浏览旧 Outline。

DONE
任何失败不得留下混合 schema。迁移测试使用 fixture 或复制 Vault，禁止把唯一真实 Vault 当首测。

失败与恢复矩阵

先保护资产，再恢复便利层。

失败层检测安全响应恢复来源
Paper 输入模型验证拒绝写入并保留表单作者修正输入
外部文件改动页面打开时保存的 source bytes拒绝覆盖磁盘外部版本 + 人工决定
单 Paper 损坏index rebuild parse error隔离 error，继续其余文件Markdown 手工修复 / 备份
Index 丢失/坏结构版本检查从 cache 重建Paper Markdown
Card state 丢失/坏JSON/类型/范围检查使用第 1 张或有效末张无需恢复
迁移失败checkpoint / staged verification不切换活动 Vault原 Vault + 外部完整备份
Provider 冲突未知/重复文件不自动合并操作系统与作者判断
