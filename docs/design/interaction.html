01

交互契约

所有核心操作共享的五条规则。

显式

保存、重命名、删除、恢复、迁移都有明确动作。

可解释

反馈说明文件、索引或设备状态中哪一层改变。

保留输入

错误不清空表单，不覆盖磁盘旧版本。

可退出

危险动作可取消；取消不产生副作用。

有限

keikeu 到外部编辑器为止，不继续追踪正文。

03

Flashcard：有限上下文

Summary 永远是第 1 张；每条 Highlight 增加一张。

FLASHCARD · 1 / 3

合成 Summary：这里只展示整体火种。

只读边界

  • 位置写入设备本地状态，不进入 Paper 或索引。
  • 状态缺失、损坏或越界时安全回到有效卡片。
  • Highlight 卡可临时展开当前 Summary，但不预览相邻卡。
  • “返回 Paper”保留作品上下文；侧栏直接进入可能没有当前 Paper，这是已记录 P2。
没有：编辑、完成、跳过、消费、正文输入、进度百分比。

05

v0.1 迁移：高风险手术

确认页只描述动作；纯 Python core 执行全部资产变更。

只读预检

列出 Cache 数量、旧 Outline 数量和阻断问题。

NO WRITE
作者显式确认

取消即停止；未勾选时迁移按钮禁用。

GATE
Vault 外完整备份

备份不进入活动索引或同步目录。

BACKUP
外部 staging 转换与逐个验证

任何 Cache 失败都不切换活动 Vault。

STAGE
原子切换与报告

全部 Paper 验证后才移除活动旧 Outline。

COMMIT
P0 停止规则：疑似数据丢失、静默改写或半迁移时立即停止写入和重复实验；保护原 Vault，只记录脱敏步骤。
06

错误与恢复

错误消息必须说明未发生什么，以及下一条安全路径。

情况立即行为资产保证下一步
Summary 为空字段旁阻止保存旧文件不变补 Summary 或离开
Paper 被外部删除/移动拒绝保存不重建假文件回 Library 刷新
Paper 被外部修改拒绝覆盖外部字节不变重新打开并人工决定
代号冲突拒绝新建/rename/restore两边都不覆盖新代号或取消
单 Paper 损坏索引记录 error其他 Paper 继续外部文本工具检查
本地 card state 损坏忽略并回第 1 张Paper 不变继续使用
Provider 冲突副本按未知文件处理不自动合并文字作者在系统层处理
07

外部编辑器交接

这是产品边界，不是缺失功能。

Paper整理已有灵感
Flashcard只读聚焦锚点
OS open / share把 Markdown 交给系统
Prose editor正文留在外部；keikeu 不监控
成功不是 keikeu 生成正文文件，而是作者能清楚地从准备与聚焦转入自己选择的正文工具。
