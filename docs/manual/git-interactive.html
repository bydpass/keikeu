观察不会改变仓库

先确认分支、脏文件和 ahead/behind。看懂现场之前，不编辑、不暂存，也不整合历史。

    脏工作树必须先列出所有文件、重叠和混入风险；本页不会替你授权后续写操作。

    keikeu 的短路径

    每一步的输出，都是下一步的输入边界。

    1. 观察确认分支和全部脏文件；有未知修改就停下确认。
    2. 限定一个 Phase、一条 `codex/` 分支、一个可独立审查的目标。
    3. 实现与检查读调用者，做最小修改，运行与风险相称的检查。
    4. 精确暂存只选已审查路径，再读 staged diff 与 status。
    5. 交接报告 HEAD、状态、证据和风险；commit 与远端动作等待明确授权。

    权限不是命令参数

    命令是否可用，与当前任务是否授权，是两件事。

    Readstatus、diff、log、show 通常只读，用来收集证据。
    Local write编辑、add、commit、switch、merge 都会改变本地状态，必须符合任务范围。
    Shared writepush 与 remote 修改会改变共享状态，必须得到明确批准。
    Destructivehard reset、clean、强删分支、破坏性 restore、改写历史都需要精确目标与明确批准。
