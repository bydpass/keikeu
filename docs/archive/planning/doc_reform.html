把文档堆，改造成一张能跑的项目地图。

Phase 8 完成后，进入 Phase 8.5：文档重构。目标只有三个： MD 瘦身、设计与架构具象化、Agent 冷启动不迷路。 这轮不扩产品功能，不重写业务代码，不让新工具反客为主。

主目标：项目地图化 HTML-first / MD-thin AGENTS.md 单一职责 Graphify = 审计层 源码与测试 = 运行事实
00

总决策

这不是“换一种文档格式”。这是一次仓库权力重分配。

核心句：HTML 负责让系统可见、可点、可核验；MD 负责目标、边界、纪律和索引；源码与测试负责证明系统真实存在。

要删掉什么

跨文件重复、长篇 UI 描述、目录考古、写给过去自己的流水账、混进 AGENTS.md 的产品百科。

要留下什么

唯一真相源、可执行视觉样张、可导航架构图、短而硬的产品契约、能直接用于审查的纪律。

最终形态

打开 README 能找到地图；打开地图能找到规范；打开规范能跳到源码；改完源码能跑测试验证。

明确不做

不把所有内容塞进一个超级 HTML；不让 Graphify 输出替代人工决策；不为“文档漂亮”拖慢 Phase 9。

01

四条铁律

后续任何文档新增、删除、移动，都先过这四关。

一条信息，一个家

同一事实只允许一个权威文件。其他地方只放链接和一句摘要，禁止复制粘贴形成多版本宇宙。

能展示，就别描述

颜色、间距、状态、响应式、交互结果进 HTML。MD 不再写“按钮大概浮起一点”这种概率云。

规范与事实分层

NORMATIVE目标与纪律由人工维护；OBSERVED源码行为由测试与扫描工具观察。两者冲突时必须处理，不准装死。

地图优先于篇幅

文档入口先告诉 agent “去哪里找”，再讲内容。一个短地图胜过三份互相抄袭的长说明。

04

HTML / MD 分工边界

HTML-first 可行，HTML-only 会翻车。边界必须写死。

design.html颜色、排版、间距、组件、状态、响应式、动效、无障碍、平台降级。
interaction.html创建、编辑、关系选择、删除、封存、导出、错误与撤销。
architecture.html模块、依赖、数据流、文件生命周期、错误恢复、源码锚点。
⇄
SPEC.md用户、问题、范围、非目标、实体、产品验收。
RULES.mdlocal-first、无静默数据损失、Markdown canonical、明确删除等。
PROJECT.md项目坐标与导航，不复制规范正文。

HTML 必须具备

可离线打开；无需构建；有目录；可搜索；组件可操作；关键节点链接到真实源码；打印仍可读。

HTML 不得承担

完整产品历史、会议流水账、Agent 性格、每次迭代状态、无法验证的宏大愿景。

Graphify 与这套思想的关系

高度相关，但它处在“观察现实”这一侧，不能坐上产品总设计师的位置。

人工规范图

我们想让系统怎样存在

architecture.html、design.html、SPEC.md表达目标、边界与判断。

Graphify 观察图

仓库实际上怎样连接

它从代码、文档和图像抽取节点与边，输出可查询图与交互 HTML，用来发现“嘴上说分层，代码里全耦合”这种事故。

差异审计

两张图对撞

期望图与现实图不一致时，只允许三种结论：改代码、改规范、登记临时偏差。禁止悄悄共存。

人工规范

SPEC / RULES / architecture.html

代码实现

src / tests / assets

Graphify 扫描

AST + docs links + inferred edges

生成观察图

graph.html / graph.json / report

漂移审计

修代码、修图或写 ADR

关键冲突：Graphify 提供面向 Codex 的安装方式，会写入 AGENTS.md。本项目禁止让插件长期占领 AGENTS。优先使用用户级 skill 或手动 CLI；生成物隔离到 docs/generated/graphify/。

试点规则

先手动运行一次，只扫描 src/、tests/ 与核心 docs。输出稳定、噪声可控后，再考虑 --update 或 git hook。

权力限制

Graphify 输出全部标为 generated。它可以证明关联、暴露孤岛、帮助查询；它无权修改产品目标、命名、交互纪律和架构意图。

10

最终判断

这轮重构的成功标准，和“文件变少”没直接关系。

成功：仓库中的每类知识都有唯一坐标；人和 agent 都能沿地图抵达源码；生成图能揭露现实漂移；下一阶段开发速度提升。
失败：把三份肥 MD 换成三份肥 HTML；Graphify 又向 AGENTS 塞一套指令；地图很漂亮，改功能时仍靠 grep 和聊天记忆。

仓库要成为一座有门牌、有地图、有监控的建筑。别再当文档乱葬岗。
