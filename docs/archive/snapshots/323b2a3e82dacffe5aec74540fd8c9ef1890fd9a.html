我们依次解决了什么

症状原因处理
找不到 main.py 项目采用 src/ 布局，Flet 默认在应用根目录寻找入口。 在 pyproject.toml 声明 [tool.flet.app] path = "src"，并增加薄入口 src/main.py。
构建后只有白屏 Python 包中的 Flet 0.86 与生成的 Flutter 客户端 0.85.3 协议不匹配；启动代码还无条件调用桌面窗口 API。 将 Flet 精确锁定为 0.85.3，并让窗口尺寸配置只在桌面平台执行。
红屏与黄黑溢出条 NavigationRail(elevation=0) 触发 Flutter 断言；桌面侧栏宽度也不适合手机。 移除无效 elevation；桌面保留 NavigationRail，iOS 改用底部 NavigationBar。
内容顶到灵动岛 移动端正文没有纳入系统安全区。 使用 SafeArea 包裹移动端正文，同时保留系统底部导航区域。
文件库只有底栏、正文消失 外层滚动容器中嵌套了 expand=True 的卡片与内部滚动列表，在无界高度下无法正确布局。 移动端只保留一个外层滚动区，取消内部 expand/scroll，并移除 420/360 像素固定宽度。
Xcode 27 beta 构建兼容问题 生成工程的最低 iOS 版本和旧式 lipo 调用与 beta 工具链存在兼容摩擦。 在生成目录调整部署目标，并使用临时兼容 shim 完成本轮构建。

进入源码的持久改动

  • pyproject.toml：锁定 Flet 0.85.3，并声明 Flet 应用目录为 src。
  • src/main.py：提供打包工具可发现的最小启动入口。
  • src/keikeu_app/main.py：区分桌面与移动平台；移动端使用底部导航、安全区和较小边距。
  • src/keikeu_app/pages/library_page.py：移动端采用单滚动区域和自适应宽度。
  • tests/test_app_pages.py、tests/test_migration_page.py：覆盖 iOS 导航、安全区、Vault 选择器与文件库布局。
重要边界： build/flutter/ios 内的 Xcode 27 调整和临时 lipo shim 属于生成环境补丁， 没有进入 Git。删除 build/、重新生成工程或清理临时目录后，可能需要重新应用。

以后如何快速重建

从仓库根目录执行。以下变量避免把个人路径写死在项目中：

export KEIKEU_FLUTTER_BIN="$HOME/flutter/3.41.7/bin"
export DEVELOPER_DIR="/Applications/Xcode-beta.app/Contents/Developer"

PATH="$KEIKEU_FLUTTER_BIN:$PATH" \
  .venv/bin/flet build ios-simulator --yes

启动目标模拟器后，安装并运行生成的应用：

export KEIKEU_SIMULATOR_UDID="<目标模拟器 UDID>"

xcrun simctl install "$KEIKEU_SIMULATOR_UDID" \
  build/ios-simulator/keikeu.app

xcrun simctl launch "$KEIKEU_SIMULATOR_UDID" \
  com.flet.keikeu

查看可用模拟器 UDID：

xcrun simctl list devices available

这是一次性部署吗？

不是一次性能力，但每次源码变化后需要重新构建和安装。

  • 源码中的入口、Flet 版本锁定和响应式 UI 是持久的，提交后不会因重启消失。
  • keikeu.app 是可丢弃构建产物；重新构建会产生新版本。
  • 模拟器中的安装副本不会自动同步源码；重新安装同一 bundle ID 会覆盖旧副本。
  • 模拟器应用的本地状态可能在覆盖安装时保留；需要全新状态时应谨慎卸载，先确认没有需要保留的测试数据。
  • 生成工程补丁目前不耐 build/ 重建，这是后续把部署流程脚本化时最值得消除的手工步骤。

继续开发 macOS 本体怎么办？

继续按原来的 macOS 开发方式运行即可。平台分支只改变壳层布局：

  • macOS 仍使用宽屏 NavigationRail 和桌面窗口尺寸。
  • iOS 使用底部导航、自适应宽度与安全区。
  • 核心 keikeu_core、Markdown Vault 和 Paper/Flashcard 数据流仍由两端共用。
  • 日常开发优先在 macOS 快速迭代；达到一个可测试节点后，再重建模拟器检查窄屏与触控流程。

仍未覆盖的风险

  1. 本轮只验证模拟器，没有验证 Apple 开发者签名、真机安装、后台恢复或 App Store 打包。
  2. Xcode 27 与 macOS 27 均为 beta 环境，后续 beta 版本可能改变构建行为。
  3. 文件库仅验证了页面显示和导航，没有在真实作者 Vault 上重复破坏性操作。
  4. 自动测试证明已覆盖的代码行为，不等同于 Phase 8 的真实作者产品验收。
