## 3. 关键差距

| 当前 v0.1 | Road v0.2 目标 | 所属 Phase |
|---|---|---|
| `Cache` + 四种创作状态 | `Paper`，无创作状态 | 1 |
| 标题 + 原始灵感 + 临时备注 | 代号 + 初稿副本 + Summary + Highlights + Tags | 1 |
| Cache/Outline 双 schema | Paper v2 单一活动 schema | 1、6 |
| Index `caches/outlines` | Index v2 `papers/errors` | 3 |
| `.trash/cache` + `.trash/outlines` | `.trash/cache` + 应用内恢复 | 3、6 |
| Cache 编辑器 | Paper 编辑器 | 4 |
| Outline 编辑器和关系 picker | 主流程不可达；Pre-Advance 后置 | 6 |
| 无 Flashcard | Summary-first Flashcard + 本地位置 | 5 |
| 文本路径 vault picker | 系统目录选择 + 普通/iCloud Drive smoke | 7 |

## 4. 高风险顺序

1. 先建立 Paper v2 I/O，不能让 GUI 自己拼 Markdown。
2. 在删除旧 Cache reader 前完成迁移器与失败注入测试。
3. 先验证全部 Cache 转换，再删除活动 Outline。
4. Outline 退休必须发生在迁移能力稳定之后。
5. iCloud Drive 等目录只按普通文件路径处理；不得顺手引入 provider API。
